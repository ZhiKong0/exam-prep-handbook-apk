# Chapter 1 全量逐选项解析复盘

## 当前解析通病
- 旧版短解析常能告诉读者“答案是什么”，但不一定把错误选项逐个拆开，小白知道结论，不一定知道排除法怎么用。
- 判断题以前更像一句话纠错，缺少 `T / F` 两个选项各自为什么对或为什么错的明示。
- 填空题容易只给标准答案，不解释为什么一定要填这个词、为什么不能随便换成近义表达。
- 知识点展开有时会偏成整章复述，离本题题干和本题干扰项有点远。

## 这次采用的写法原则
- `quickExplanation` 固定保留 `题眼 / 答案(或判断) / 理由 / 易错`，让后续脚本或 UI 还能继续解析。
- `single / multi / tf` 全部补成逐选项判断，每个选项单独一行，明确写“对/错”和本题原因。
- `blank` 全部补成逐空说明，优先解释“为什么这空必须填这个标准词”。
- `knowledgeDetail` 严格只保留 `核心知识点 / 题目变形 / 知识拓展` 三段，避免再滑回长模板。
- 尽量把解释贴在题干关键词和选项差异上，而不是泛讲大段百科背景。

## 抽样 5 题前后对比观察
### 2-1 `bps`
- 之前：能看出正确答案是 `Bits per Second`，但 A/C/D 错在哪，得自己补脑。
- 现在：A、B、C、D 四项都单独写了，`Byte / bit / Baud / 数量级词` 一次说清。
- 观察：这类单位题补齐排除理由后，小白更容易秒选。

### 2-15 `2000 band, 每码元 4 bit`
- 之前：有公式，但更像只告诉你“乘一下”。
- 现在：把单位错、少乘 4、把 band 当 bit/s 这些典型误选都逐项拆开。
- 观察：计算题的关键不是只给结果，而是让人知道错法长什么样。

### 3-6 `协议三要素`
- 之前：已有多选结论，但 UI 侧落地时不一定能稳定看出每项的对错边界。
- 现在：A/B/C/D 每项都单独成行，`接口` 为什么不算三要素说得更明确。
- 观察：多选题只要把“入选标准”和“排除标准”讲清，小白就不容易少选或多选。

### 1-30 `C/S 和 B/S`
- 之前：能知道题干错了，但 B/S 和 C/S 为什么不是并列大类，说得还不够像排错过程。
- 现在：直接写出 `T 错 / F 对`，并把 `B/S 是 C/S 在 Web 场景中的一种形式` 放到错点上。
- 观察：判断题只纠正一句话不够，最好把两个判项都亮出来。

### 4-64 `数据报 和 虚电路`
- 之前：能背答案，但“为什么这里填数据报，不是写成别的词”不够明确。
- 现在：补了逐空解释，并明确这是“分组交换两种实现方式”的固定搭配。
- 观察：填空题一旦补上“标准词意识”，做题稳定性会高很多。
