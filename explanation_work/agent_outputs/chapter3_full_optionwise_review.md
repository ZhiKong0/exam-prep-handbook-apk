# Chapter 3 全量逐项解析说明

范围：`chapter == "3. 数据链路层与局域网"`，共 72 题。
题型分布：判断 29、单选 13、多选 16、填空 14。
输出目标：让小白看完就知道这题为什么对、为什么错，而不是只背答案。

## 当前解析通病

1. 旧短解析往往只说“为什么选它”，但单选和判断题对错误选项的拆解不够直白，小白容易只记结论。
2. 多选题虽然常有一句总判断，但未必把每个选项都单独落成“对/错 + 本题原因”，复习时不够像答题卡旁边的小老师。
3. 填空题容易只报标准答案，没有把每一空和题干里的限定词一一对上，记忆点容易散。
4. 详解版知识点有时偏长，手机端刷题时很容易把“本题题眼”淹没掉。
5. 第 3 章里最容易混的正是设备层次、地址层次、编码/检错、CSMA/CD/CSMA/CA、Hub/Switch/VLAN；如果不贴着题干说，学生很难真正排除干扰项。

## 这次采用的写法原则

1. `quickExplanation` 固定保留 `题眼 / 答案(或判断) / 理由 / 易错`，同时补上 `选项判断` 或 `空格判断`，让每题都能直接看到逐项结论。
2. 单选和多选都把每个选项拆成单独一行，明确写“对/错”，避免把错误项压缩成一句笼统的“排除/易混”。
3. 判断题不只写“错”，而是把 T/F 两个选项都写明，并尽量指出题干中的错词或错位概念。
4. 填空题按“第几空”逐格解释，让学生知道这个空为什么填这个词，而不是只知道最终答案串。
5. `knowledgeDetail` 严格压成 `核心知识点 / 题目变形 / 知识拓展` 三段，避免长篇模板化铺陈，同时继续保留本题相关的辨析和扩展。

## 抽样 5 题前后对比观察

- `1-2`：旧稿已经能指出“流量控制”写错了，但没有把 T/F 两个判断选项分别落地；新稿直接写出 `T 错 / F 对`，更像在带着学生判题。
- `2-17`：旧稿能说出正确答案是曼彻斯特编码，但错误项还是合并在一句“排除/易混”里；新稿把 A/B/C/D 四项逐个拆开，小白一眼能看出“循环码是检错，不是编码”。
- `3-38`：旧稿已有较强的逐项判断雏形；新稿把每项正式落成独立行，并把“检错不等于纠错”固定下来，复习时更便于扫读。
- `4-12`：旧稿知道 10BASE-T 的四个答案，但仍像一段总说明；新稿把四个空分别对应到“拓扑 / 速率 / 介质 / 信号类型”，更适合直接背题。
- `3-72`：这是典型“题干要求选错误项”的陷阱题。新稿把这个陷阱直接写进 `理由`，并分别说明 A 为什么不该选、B/C/D 为什么该选，能明显减少审题失误。
