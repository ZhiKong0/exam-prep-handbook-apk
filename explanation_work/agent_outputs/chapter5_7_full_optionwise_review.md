# 第 5-7 章逐选项解析复盘

覆盖范围：`5. 运输层`、`6. 应用层`、`7. 网络安全`，共 86 题。
题型分布：判断 19、单选 49、多选 2、填空 16。
输出文件：`chapter5_7_full_optionwise.json`。

## 当前解析通病

1. 旧解析里有不少“整章概念没少写，本题选项却没拆开”的情况。小白看完知道这章在讲什么，却还是不知道 A 为什么错、D 为什么对。
2. 运输层和应用层的计算/流程题，旧稿容易把 TCP、HTTP、DNS、FTP 的通识铺得很长，但没有直接把本题最关键的一步钉住，比如“ACK 填下一个期望字节号”“keep-alive 代表持续连接”。
3. 判断题和填空题里，原有内容有时能给出正确结论，但没有把“T/F 为什么这样判”“这个空为什么不能填近义词”说透。
4. 个别题的知识点展开会跑远，出现“题目问 UDP，解析却讲成 TCP 通识”“题目问防火墙分类，解析却泛讲网络安全”的问题。
5. 多选题数量虽然少，但旧写法容易只给出组合答案，不把“不选项”逐个判掉，学生仍会按单选思路做题。

## 本次写法原则

1. 每题的 `quickExplanation` 都固定保留 `题眼 / 答案或判断 / 理由 / 选项判断或空格判断 / 易错`，保证手机上扫一眼就能找到结论。
2. 对 `single / multi / tf`，所有选项都单独成行，明确标出“对/错”和本题原因，不再只说“其他项不匹配”。
3. 对 `blank`，每空都解释“为什么正好填这个词”，并且指出常见近义词为什么不如标准答案稳。
4. `knowledgeDetail` 只保留三段：`核心知识点`、`题目变形`、`知识拓展`，避免再散成大段百科。
5. 尽量把解释贴在题干动作上：问端口就盯端口，问查询方式就盯递归/迭代，问连接顺序就盯控制连接/数据连接，不把题目抹平成统一模板。

## 抽样 5 题前后对比观察

1. `2-67` TCP 确认号  
   旧问题：容易看完只记住“TCP 很可靠”，但不知道为什么是 1000。  
   这次：直接把 4 个选项都拆开，明确 `500`、`700`、`800` 分别错在“只算一段”“少加一段”“忘加起始序号”。

2. `2-102` HTTP 请求头判断  
   旧问题：能看出答案是 `C`，但 A/B/D 为什么成立没有逐项钉住。  
   这次：把 `GET /index.html`、`Host`、`Connection: keep-alive`、`Cookie` 四个字段各自翻成一句人话，学生能直接学会读请求头。

3. `1-15` 数字签名判断题  
   旧问题：知道“错”还不够，小白仍容易把“公钥加密”和“公钥签名”混为一谈。  
   这次：在 `选项判断` 里单独写出 `T 错 / F 对` 的原因，并把“私钥签、公钥验”压成一句可背的记忆点。

4. `4-72` 防火墙填空  
   旧问题：容易泛讲防火墙功能，空格本身却没有被钉住。  
   这次：直接说明既然后一半已经给了“代理服务器或应用网关”，前一半就必须填它的并列项“分组过滤路由器”，并指出“包过滤”这类近义词为什么不如标准答案稳。

5. `3-67` Web 服务器多选  
   旧问题：只告诉 `ACD` 还不够，学生仍会犹豫 B。  
   这次：把 A、B、C、D 四个选项都拆开，特别强调“服务器负责提供页面，不等于负责创建和编辑页面”，多选题的排除逻辑更清楚。
