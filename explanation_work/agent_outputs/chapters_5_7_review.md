# 第 5-7 章解析质量审查与个性化讲解模型

范围：`chapter` 为 `5. 运输层`、`6. 应用层`、`7. 网络安全` 的 87 题。  
目标：不直接修改题库，提炼后续批量优化 `quickExplanation` 与 `knowledgeDetail` 可复用的讲解模型。

## 总体问题

1. **通用模板套入过重**：判断题常出现“先定规则、改正方向、底层抓手”等长套话，手机阅读时题眼被冲淡。
2. **知识模型错配**：部分题目的 `knowledgeDetail` 套到了其他知识点，例如 TCP 确认号题讲 FTP，UDP 题讲复用技术，防火墙题讲路由协议。
3. **计算/流程题缺少稳定步骤**：TCP 确认号、HTTP RTT、DNS 查询次数、DHCP Discover 地址等题，需要“先拆对象，再套规则，再算/选”的固定模板。
4. **反向单选提示不够精确**：有些解析把普通单选误说成“反向题”，或只说“其他项不匹配”，没有指出每个干扰项错在何处。
5. **多选题数量少但要求高**：第 5-7 章多选只有 2 题，更应明确“为什么要多选”和“为什么不选”，避免学生按单选思维找唯一最佳项。
6. **第 7 章题量少，易被泛化**：网络安全只有 6 题，解析必须抓住公钥/私钥、数字签名、防火墙、安全协议、主动攻击这几个高频边界，不宜套整章安全概论。

## 分组讲解模型

### 运输层服务、UDP、端口与套接字

**当前模板问题**  
解析容易把“运输层端到端”讲成泛泛分层知识，或把 UDP 题套到物理层复用模型。端口、套接字、UDP 校验和这些题应短句直打概念边界。

**小白底层模型**  
网络层把分组送到主机，运输层再用端口号送到具体应用进程。UDP 是无连接、尽力而为、面向报文的运输层协议，有首部和校验和，但没有确认、重传、排序和拥塞控制。套接字通常可理解为 `IP 地址 + 端口号`，只看 IP 只能定位主机，不能定位应用。

**多选规则**  
遇到 UDP/TCP 共有功能，选“端口复用/分用、校验和”；不因 UDP“不可靠”就排除校验。遇到“可靠传输、面向连接、字节流、拥塞控制”优先归 TCP；遇到“保留应用边界、首部小、无连接”优先归 UDP。

**填空/流程模板**  
1. 先问通信对象：主机到主机还是进程到进程。  
2. 问主机标识填 IP，问进程标识填端口，问 TCP 连接端点填套接字。  
3. 问 UDP 错误项时，保留“无连接、复用/分用、检错”，排除“保障可靠传输”。  
4. 问公共服务端口时，记住 0-1023 是熟知端口范围。

### TCP 连接管理、可靠传输与流量控制

**当前模板问题**  
TCP 题目常被套到 FTP 或宽泛分层模型，导致确认号、序号、字节流、连接端点这些本题关键步骤不突出。

**小白底层模型**  
TCP 面向连接、可靠、面向字节流。序号标的是“本段数据第一个字节的编号”，确认号标的是“期望收到的下一个字节编号”，所以确认号不包含它自己。可靠性靠序号、确认、重传和窗口协同完成，流量控制主要看接收方窗口，避免把接收方压垮。

**多选规则**  
TCP 机制题应选“连接、可靠、字节流、确认、重传、流量控制、拥塞控制”；不选“保留应用报文边界”。如果题目同时问 TCP/UDP，二者都可选端口复用/分用和校验和，但只有 TCP 可选可靠交付。

**计算/流程模板**  
1. 找第一个序号 `seq`。  
2. 累加已经正确收到的数据字节数。  
3. 确认号 `ACK = seq + 已收到总字节数`。  
4. 如果题干说“确认号为 n”，解释为下一个期望字节是 n，即 n 之前已收到，不包括 n。

### TCP 拥塞控制

**当前模板问题**  
拥塞控制解析容易混入链路层或 TCP 序号题，缺少“全局网络问题”和“四个算法”的固定对照。

**小白底层模型**  
流量控制管的是发送方别把接收方撑爆；拥塞控制管的是全网别被过多分组压垮。拥塞不是单台主机的缓存问题，而是主机、路由器、链路负载共同造成的全局性能下降。教材常考四个算法：慢开始、拥塞避免、快重传、快恢复。

**多选规则**  
拥塞控制题选“全局性、网络负载、路由器队列、降低网络性能”；不选“只由接收方能力决定”。和流量控制并列出现时，用“接收方窗口 vs 拥塞窗口”区分。

**填空/流程模板**  
1. 问拥塞根据：网络中分组数过多、路由器队列增长、吞吐下降。  
2. 问四算法：慢开始、拥塞避免、快重传、快恢复。  
3. 问与流量控制区别：流量控制看接收方，拥塞控制看网络整体。

### DNS 域名系统

**当前模板问题**  
DNS 解析常被写成大段层次结构介绍，但多数题只考“谁问谁、递归还是迭代、域名到 IP、UDP/TCP 53”。

**小白底层模型**  
DNS 的核心任务是把域名解析成 IP 地址。主机通常把问题递归交给本地域名服务器：你帮我查到底；本地域名服务器再常用迭代方式问根、顶级域、权威服务器：你不知道就告诉我下一步问谁。递归强调“代查到底并返回结果”，迭代强调“返回线索，让查询者继续问”。

**多选规则**  
DNS 组成题选域名空间、域名服务器、解析器；不选 HTTP/网页服务器。运输层服务题中 DNS 可选 UDP，因为普通查询常用 UDP 53，但区域传送等场景可用 TCP，解析不要绝对化。

**流程模板**  
1. 先定位发问者：主机问本地 DNS，还是本地 DNS 问外部 DNS。  
2. 看到“原路返回最终结果”判递归。  
3. 看到“返回下一步应查询的服务器”判迭代。  
4. 计算查询次数时，按根域、顶级域、权威域逐级数，注意缓存会减少次数。

### Web、HTTP、URL 与万维网

**当前模板问题**  
HTTP 题会混入 DNS/TCP/页面对象，但解析没有稳定拆分步骤。多选题还需要解释 Web 服务器“提供页面”不等于“编辑页面”。

**小白底层模型**  
Web 访问一般是浏览器拿 URL，必要时先 DNS 解析，再用 TCP 承载 HTTP 请求/响应。HTTP 是应用层协议，本身无状态，Cookie 用来帮助站点识别用户。持续连接可以复用同一个 TCP 连接；非流水线表示仍要等上一个响应回来再发下一个请求。

**多选规则**  
Web 服务器题选“存放/提供 HTML 等 Web 文档、WWW 服务器、客户端是浏览器”；不选“服务器必须创建和编辑页面”。HTTP 题选 TCP 面向连接服务；不选 UDP 作为普通 HTTP 承载。

**计算/流程模板**  
1. 先数对象：HTML 本身算 1 个对象，每张图片/资源另算。  
2. 判断连接方式：HTTP/1.0 非持续通常每个对象建一次连接；HTTP/1.1 持续可复用连接。  
3. 判断是否流水线：非流水线按对象逐个请求-响应；流水线可连续发请求。  
4. 最后看题干是否把 DNS RTT、TCP 建连 RTT 纳入计算。

### 电子邮件：SMTP、POP3、IMAP 与 MIME

**当前模板问题**  
邮件题目前解析整体较可用，但仍需要更强调“发送/转发/收取/内容扩展”四分法，避免把 SMTP 用到所有邮件阶段。

**小白底层模型**  
SMTP 负责发送邮件：用户代理到发送方邮件服务器、邮件服务器之间转发。POP3/IMAP 负责收取邮件：用户从邮箱服务器读取。MIME 不是另一个取信协议，而是扩展邮件内容格式，让非 ASCII 文本、图片和附件可以进邮件。

**多选规则**  
SMTP 正确项可选“7 比特 ASCII 原始限制、用户代理到邮件服务器、服务器到服务器”；不选“邮件服务器到用户代理取信”。POP3/IMAP 题选读取/接收邮件，不选发送。

**流程模板**  
1. 题干说“发出/投递/服务器之间”选 SMTP。  
2. 题干说“用户从服务器取回/读取”选 POP3 或 IMAP。  
3. 题干说“附件、非 ASCII、多媒体”选 MIME。  
4. 问传输层服务时，SMTP、POP3、IMAP 通常都基于 TCP。

### FTP 文件传输

**当前模板问题**  
FTP 题解析有时能讲两条连接，但对“控制连接先建晚释放、数据连接按需建立”的时序还应更短更明确。

**小白底层模型**  
FTP 是应用层文件传输协议，使用两条 TCP 连接：控制连接传命令和响应，数据连接传文件内容或目录列表。控制连接通常连服务器 21 端口，并贯穿整个会话；数据连接只在需要传数据时建立，传完释放。TFTP 才是“小而简单”的文件传送协议，FTP 不属于很小且容易实现。

**多选规则**  
FTP 连接题选“控制连接、数据连接、TCP、控制连接先建立晚释放”；不选“控制连接传文件内容”。端口题要看主动/被动模式，教材常考服务器 21 为控制连接、20 常用于主动模式数据连接。

**流程模板**  
1. 先判断题干问命令还是文件/列表。  
2. 命令走控制连接，数据/目录列表走数据连接。  
3. 控制连接先建立，数据连接按需建立。  
4. 控制连接最后释放，数据连接传完先释放。

### DHCP、SNMP/MIB 与其他应用层协议

**当前模板问题**  
DHCP Discover 地址题被套入分层模型，MIB 填空也容易缺少“SNMP 三件套”的短记忆。

**小白底层模型**  
DHCP 解决“新主机刚接入网络时自动拿 IP 和参数”的问题。Discover 阶段主机还没有可用 IP，所以源地址是 `0.0.0.0`，为了找到服务器，目的地址用受限广播 `255.255.255.255`。SNMP 网络管理常和 SMI、MIB 成组出现，MIB 是管理信息库。

**多选规则**  
DHCP 题选“自动配置、获取 IP、广播发现”；不选手工配置或回环地址。SNMP 题把协议、信息结构和管理信息库分开，不把 MIB 当作传输协议。

**流程模板**  
1. Discover：源 `0.0.0.0`，目的 `255.255.255.255`。  
2. 看到 `127.0.0.1` 直接警惕：这是本机回环，不用于发现 DHCP 服务器。  
3. SNMP 组成填空：SNMP、SMI、MIB。  
4. 动态主机配置协议填 DHCP。

### 密码学、报文鉴别、数字签名与不可否认

**当前模板问题**  
安全解析容易泛讲“保密、完整性、认证”，但数字签名题真正考的是私钥签名、公钥验证。

**小白底层模型**  
保密性主要问“别人看不看得懂”，报文鉴别和完整性问“是不是本人发的、途中有没有被改”。数字签名用发送方私钥生成，接收方用发送方公钥验证，因此能提供报文鉴别、完整性和不可否认。公钥加密和数字签名方向相反：保密常用对方公钥，签名用自己私钥。

**多选规则**  
数字签名功能题选报文鉴别、完整性、不可否认；不选“绝对保密”作为签名本身功能。公钥/私钥方向题中，看到“用公钥签名”通常错误。

**填空/流程模板**  
1. 先判断目标：保密、鉴别、完整性、认证还是不可否认。  
2. 签名：发送方私钥签，发送方公钥验。  
3. 加密保密：常用接收方公钥加密，接收方私钥解密。  
4. 数字签名三功能：报文鉴别、报文完整性、不可否认。

### 防火墙、安全协议、VPN 与网络攻击

**当前模板问题**  
该组题量少，但当前解析有把防火墙分类套成路由知识的情况。应直接围绕标准术语和安全层次。

**小白底层模型**  
主动攻击强调攻击者主动改变、注入或破坏，如篡改、恶意程序、拒绝服务。运输层安全协议常见 SSL/TLS，SSL 是安全套接字层，TLS 是运输层安全。防火墙教材常分为分组过滤路由器和代理服务器/应用网关，前者偏按分组头部规则过滤，后者偏应用层代理检查。

**多选规则**  
攻击类型题区分主动与被动：窃听偏被动，篡改/恶意程序/拒绝服务偏主动。防火墙分类题保留标准术语，不把“路由器”“网关”泛化替代完整答案。

**填空/流程模板**  
1. 主动攻击三类常填：篡改、恶意程序、拒绝服务。  
2. SSL 填“安全套接字层”，TLS 填“运输层安全”。  
3. 防火墙分类：分组过滤路由器 + 代理服务器/应用网关。  
4. VPN 题先抓隧道、加密、认证，再看协议位于哪一层。

## 改写建议

1. `quickExplanation` 控制在 3-5 行：题眼、答案、核心理由、易错点。
2. `knowledgeDetail` 控制在 6-10 行：先给本题底层模型，再给步骤或选项拆解。
3. 单选反向题必须写清“题目要选错误项”，普通单选不要误标为反向题。
4. 多选题固定写“应选 A/C/D，因为……；不选 B，因为……；本题为什么多选：这些是并列成立的特征”。
5. 填空题优先给标准术语，并说明近义词为什么不如标准答案稳。
6. 计算题固定使用“公式/规则 + 代入 + 结论”，不要只给选项。
7. 协议流程题固定按阶段拆：DNS 递归/迭代、HTTP 对象和 RTT、FTP 控制/数据连接、DHCP Discover 地址、邮件发送/收取。
